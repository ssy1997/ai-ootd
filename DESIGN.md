### 3.2 对话流程设计

#### 场景一：根据天气推荐
```
AI: 你好呀！我是你的AI穿搭师 🤗 今天想怎么穿呢？

[用户选择：根据天气]

AI: 今天北京天气晴朗，15°C，很适合春装呢 ☀️
    你今天有安排什么场合吗？

[用户选择：约会]

AI: 约会呀！那我给你推荐一套温柔又精致的搭配吧 💕
    看看这身搭配怎么样？

[展示穿搭图片]

AI: 这套搭配你觉得怎么样？
    还是需要我帮你调整一下？

[用户选择：喜欢]

AI: 太好啦！那我就帮你记录下来啦 ✨
    还有其他需要我帮忙的吗？
```

#### 场景二：根据场合推荐
```
AI: 你好呀！我是你的AI穿搭师 🤗 今天想怎么穿呢？

[用户选择：根据场合]

AI: 好的，告诉我今天是什么场合？

[用户选择：约会]

AI: 约会呀，那今天的心情是怎么样的呢？

[用户选择：开心]

AI: 开心去约会，那我们穿得漂漂亮亮的吧！
    给你推荐这套...

[展示穿搭图片]
```

#### 场景三：参考穿搭图推荐
```
AI: 你好呀！我是你的AI穿搭师 🤗 今天想怎么穿呢？

[用户选择：参考穿搭图]

AI: 好的，请发给我你喜欢的穿搭图片吧～

[用户上传穿搭图]

AI: 收到啦！让我看看... 🤔
    这是一套简约清爽的搭配呢
    我从你衣橱里帮你找找相似的...

[AI分析中...]

AI: 找到啦！根据这张图的风格，
    我给你准备了这几套...

[展示多个穿搭选项]

AI: 你更喜欢哪一套呢？
```

### 3.3 快捷回复类型

| 类型 | 说明 | 示内容 |
|------|------|--------|
| 天气 | 根据天气推荐 | 根据今日天气推荐 |
| 场合 | 选择场合 | 约会、通勤、运动、旅行 |
| 心情 | 选择心情 | 开心、沉稳、活泼、慵懒 |
| 操作 | 交互操作 | 喜欢、不喜欢、换一套一套、保存 |
| 上传 | 上传图片 | 上传参考穿搭图 |
| 结束 | 结束对话 | 好的，谢谢 |

## 五、数据模型设计

### 5.1 服饰

```typescript
interface ClothingItem {
  _id?: string;              // 云数据库 ID
  openid: string;             // 用户 OPENID
  imageUrl: string;           // 图片URL（云存储）
  name: string;              // 服饰名称
  category: Category;         // 品类
  color: string[];            // 颜色数组
  season: Season[];           // 季节数组
  occasion: Occasion[];       // 场合数组
  style: Style[];             // 款式数组
  material: string[];        // 材质数组
  note: string;               // 备注
  useCount: number;           // 使用次数
  lastUsedAt?: Date;          // 最后使用时间
  createdAt: Date;           // 创建时间
  updatedAt: Date;           // 更新时间
}
```

```typescript
interface Outfit {
```

## 九、UI/UX注意事项

### 9.1 颜色方案
- 主色：#6C5CE7（深邃紫）
- 辅色：#8A7FB0（紫灰）
- 强调色：#FF6B9D（粉红，用于收藏、喜欢）
